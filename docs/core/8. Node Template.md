# Markdown Body Equations

Display equations in fenced code blocks. KaTeX renders them in the standards viewer.

```markdown
The thickness follows eq. (2):

```
$$
t_m = t + c
\tag{2}
$$
```
```

You can also use a single-line fence: `` `t_m = t + c \tag{2}` `` inside a fenced block without `$$` wrappers.

Use `\tag{n}` for equation numbers. Bare `$$ ... $$` outside a fenced block is not parsed; keep the fence.

---

# Markdown Body Cross-References

In the markdown body (below the frontmatter `---`), link to other nodes with standard markdown links and custom URL schemes:

```markdown
Internal pressure thickness is defined in [§304.1.2](node:B313-304.1.2).
Quality factor **E** comes from [Table A-1A](table:asme_b31.3_A-1A).
Weld joint strength reduction is defined in [§302.3.5(e)](node:B313-302.3.5/e).
```

- **Link text** — user-facing label (paragraph, table name, etc.)
- **`node:<node_id>`** — engineering node id (same as YAML `depends_on` / `references`)
- **`node:<node_id>/<subsection_id>`** — focused subsection view (YAML `subsections[].id`, e.g. `e` for §302.3.5(e))
- **`table:<table_id>`** — pack-scoped table id from the standards database (e.g. `asme_b31.3_A-1A`, `asme_b31.3_table_304_1_1`)

The desktop app renders these as clickable references that open content in a right-panel tab. Keep YAML `depends_on` and `references` for workflow and graph logic; use `node:` / `table:` links in prose for human-readable cross-references.
