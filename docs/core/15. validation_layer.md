# Validation Layer Design

## Purpose

**`ValidationEngine`** (`engine/validation/validation_engine.py`) is the compliance gate: *can this planned operation proceed?*

**`RuleEngine`** (`engine/rules/rule_engine.py`) evaluates narrow runtime expression conditions only — not the full compliance layer.

---

# Owns / must not own

| Owns | Must not |
| --- | --- |
| Plan-level compliance (`validate_plan`) | Workflow selection |
| Per-node compliance (`validate_node`) | Formula execution |
| Unit, input, engineering, dependency validators | Graph expansion or path generation |
| Override recording (where implemented) | Lookup resolution |

---

# Validation timing

| Timing | What | Where |
| --- | --- | --- |
| **Input acceptance** | Parsing, type, unit validity at store time | Partial — **deferred** where not wired to input path |
| **Plan-level** | Whether `ExecutionPlan` may proceed | `validate_plan()` before `Executor.execute_plan()` |
| **Per-node** | Whether node may run / outputs valid | `validate_node()` before each node in Executor |

`RuleEngine` is not a substitute for any row above.

---

# Compliance status mapping

| Status | Behavior |
| --- | --- |
| `PASS` | Proceed |
| `PASS_WITH_WARNING` | Proceed; warnings stored |
| `INCOMPLETE` | Pause (`awaiting_input`); missing inputs or dependency outputs |
| `FAIL` | Stop / invalidate; hard rule violation |

Executor maps `FAIL` → `INVALIDATED` where implemented.

---

# Overrides

Only findings explicitly classified as **overrideable** may continue after explicit user decision.

| Outcome | Behavior |
| --- | --- |
| `FAIL` / non-overrideable constraint | Stop or invalidate — user confirmation must **not** silently become a warning |
| Overrideable finding | May proceed with audit record (rule id, severity, reason, user decision, timestamp) |

`ValidationEngine.record_override()` and `task.outputs["validation_overrides"]` support partial recording.

**Implementation gap:** CLI override UI deferred; not all paths distinguish overrideable vs hard `FAIL` at input time.

---

# Lookup-derived values

Do not treat lookup outputs (E, W, Y, S, allowable stress, OD from NPS) as user-confirmed defaults. Users provide lookup **keys**; resolution follows lookup contracts. Policy: [`docs/rules.md`](../rules.md) §16.

---

# Pipeline

```
ExecutionPlan (from Graph Engine)
↓
ValidationEngine.validate_plan()
↓
Executor.execute_plan() — per node:
  ValidationEngine.validate_node()
↓
NodeRunner (when allowed)
↓
task.outputs["_validation_trace"]
```
