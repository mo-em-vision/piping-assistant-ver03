# Execution Layer Design

## Purpose

The Execution Layer (Kernel / **Executor**) schedules and executes an already resolved **`ExecutionPlan`** after the Planner and Graph Engine have expanded the active subgraph and `ValidationEngine.validate_plan()` has allowed the plan.

It executes engineering truth; it does not decide engineering truth.

---

# Owns / must not own

| Owns | Must not |
| --- | --- |
| `Executor.execute_plan()` on resolved plan | Workflow selection, intent interpretation |
| Formula, lookup, validation-rule runners | Raw-graph dependency discovery (canonical path) |
| Per-node trace and intermediate values | Branch selection during expansion |
| Unit handling in calculation path | Compliance gate (delegates to `ValidationEngine`) |

---

# Canonical execution flow

```
ExecutionPlan received (from Graph Engine, after validate_plan)
↓
Per node: ValidationEngine.validate_node()
↓
Inject values from Facts / prior node outputs
↓
Execute (equation / lookup / validation_rule runner)
↓
Store trace → next node
```

Dependency order is **precomputed in the plan** — the Executor does not perform graph discovery in the canonical runtime path.

---

# Raw graph execution (legacy / testing only)

> **Not the canonical runtime path.** Some tests may pass `nodes + dependencies` directly for replay or isolation. Production flow always uses a resolved `ExecutionPlan` from the Graph Engine.

---

# Node execution contract

Each node execution records: node id, inputs, outputs, warnings, trace metadata. Intermediate values are preserved unrounded until presentation.

Lookup outputs (E, W, Y, S, allowable stress, etc.) are derived at runtime — not user-confirmed defaults. See [`docs/rules.md`](../rules.md) §16.

---

# Missing inputs during execution

If required inputs are absent, execution pauses (`ExecutionStatus.AWAITING_INPUT` / `NodeExecutionStatus.AWAITING_INPUT`) and control returns to the Planner/input collection path — the Executor does not prompt users directly.

---

# Overrides

`FAIL` from `ValidationEngine` stops affected execution. The Executor does not silently convert hard failures to warnings. Overrideable findings require explicit recorded user decision — see [`15. validation_layer.md`](15.%20validation_layer.md).
