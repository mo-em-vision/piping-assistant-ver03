# AI Agent Design

## Purpose

AI agents provide navigation and communication intelligence. They do not define engineering truth, invent missing requirements, or author deterministic workflow prompt text.

> Agents decide what to do next, but never what the truth is.

---

# Agent inventory

| Agent | Module | Deterministic path | LLM fallback |
| --- | --- | --- | --- |
| Intent | `intent_agent.py` | `engine.router` keyword match | Unmatched requests |
| Planner | `planner_agent.py` → `engine/planner/planner.py` | Graph-active navigation | Ambiguous intents |
| Input | `input_agent.py` | `NavigationPlan.phase_missing` only | When no structured prompt exists |
| Context | `context_agent.py` | Off-topic detection; no LLM during active task | Unrelated input, no active task |
| Synthesis | `synthesis_agent.py` | Markdown fallback from `ReportData` | Report presentation prose |
| Routing | `routing_agent.py` | Standard routing heuristics | Ambiguous standards |

Environment: `OPENAI_API_KEY`, optional `OPENAI_MODEL`. As-built: [`ai/agents/README.md`](../../ai/agents/README.md).

# Presentation ownership

| Concern | Owner | Not agents |
| --- | --- | --- |
| Intent ambiguity, context switching | LLM agents | — |
| Active workflow parameter questions | `engine/messaging/` | Agents |
| Traversal narration | `GuidanceResolver` | Agents |
| Live UI response assembly | `ResponseComposer` | Agents |
| Optional report wording | `SynthesisAgent` / `engine/reports/presentation.py` | Engineering values |

---

# LLM usage boundary (chat)

| When | LLM? |
| --- | --- |
| First message — classify intent | Yes, if router cannot match |
| Active task — expansion, path, parameters | **No** |
| Active task — off-topic message | Yes (context switch) |
| Ambiguous workflow / multiple roots | Yes |

Deterministic prompts: `engine/messaging/step_prompt.py`, `formula_parameter_prompt.py`, `ai/input_extractor.py`.

---

# Agents must not

- Execute formulas or modify engineering values
- Bypass `ValidationEngine` outcomes
- Invent graph-active requirements or deterministic phase prompts
- Silently convert `FAIL` to warnings on overrides

Override policy: [`15. validation_layer.md`](15.%20validation_layer.md).

# Example — intent

User: "Design pipe thickness for refinery service"

→ `pipe_wall_thickness_design`, domain piping, workflow [`workflows/pipe-wall-thickness.yaml`](../../workflows/pipe-wall-thickness.yaml).

Graph expansion and Planner determine missing fields — not the LLM during deterministic phases.

---

# Final principle

Graph defines truth. Rules define validity. Agents navigate. Calculations produce results. Reports explain.
