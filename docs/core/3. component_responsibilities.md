# Component Responsibilities

## Purpose

Single source for **who owns what** between system components. Other docs link here instead of duplicating the pipeline diagram.

Core principle:

> Planner orders the next graph-active work; graph expansion determines active requirements. Neither alone defines engineering truth.

# Component reference

| Component | Owns | Must not |
| --- | --- | --- |
| **IntentAgent** | User language → structured intent | Engineering requirements, prompt copy |
| **Planner** | Workflow selection/ranking, phase order, next graph-active unresolved requirement | Active branches, formulas, compliance decisions, user-facing prompt text |
| **Graph Engine** | `expand_workflow`, reachability, branch activation, dependencies, execution ordering | Formulas, lookups, validation, calculations, traversal narration |
| **RuleEngine** | Narrow runtime expression conditions delegated during node execution | Graph edge `when` during expansion; full compliance gate |
| **ValidationEngine** | Plan-level and per-node compliance (`validate_plan`, `validate_node`) | Path selection, calculations |
| **Executor** | Schedule and run resolved `ExecutionPlan` | Raw-graph dependency discovery (canonical path), branch selection |
| **State Manager** | Task lifecycle, persistence, invalidation, execution history | Engineering meaning |
| **Reference Resolver** | Paragraph/table/cross-ref resolution | Calculations |
| **Messaging** | Deterministic parameter and step prompt copy | Which parameter is active |
| **GuidanceResolver** | Traversal narration from `presentation/guidance/workflows/` | Parameter prompts |
| **ResponseComposer** | Merge guidance + messaging + warnings for live UI | Engineering truth |
| **Report Generator / ReportData** | Deterministic report from task state + traces | Live workflow prompts |
| **AI Presentation** | Optional report wording polish | Values, formulas, PASS/FAIL |
| **Event Logger** | Audit events for replay | Engineering decisions |

Condition ownership: graph expansion evaluates edge `when` and gates via `engine/graph/conditions.py`. `ValidationEngine` is the compliance gate. Components do not independently re-evaluate the same edge condition for different purposes.

# Graph Engine

Authored standards and knowledge nodes contain engineering rules. The Graph Engine is authoritative for graph structure, expansion, and execution ordering on the active path.

- Loads and traverses compiled pack graphs
- Produces active subgraph and `ExecutionPlan` inputs
- Does not execute formulas, perform lookups, validate compliance, or calculate results

Detail: [`14. graph_engine_design.md`](14.%20graph_engine_design.md). Expansion policy: [`docs/rules.md`](../rules.md) §13.

---

# Validation Layer

`ValidationEngine` answers: *is this allowed?*

| Timing | Where |
| --- | --- |
| Plan-level compliance | `validate_plan()` before `Executor.execute_plan()` |
| Per-node compliance | `validate_node()` before each node run |
| Input acceptance at store time | Partial — deferred where not wired |

`FAIL` and non-overrideable constraints stop or invalidate execution. Only explicitly overrideable findings may continue with recorded user decision.

Detail: [`15. validation_layer.md`](15.%20validation_layer.md).

---

# Execution Layer

Executes a **resolved** `ExecutionPlan` after validation allows it. Does not select workflows, interpret intent, or discover dependencies in the canonical runtime path.

Detail: [`13. execution_layer_design.md`](13.%20execution_layer_design.md).

---

# Planner Layer

Selects/ranks workflow candidates and orders the next graph-active missing requirement in phase order. Consumes graph expansion output (`required_user_inputs()`, `NavigationPlan`).

Prompt copy is **not** on the Planner — see `engine/messaging/`. Detail: [`11. planner_layer_design.md`](11.%20planner_layer_design.md).

---

# Agent Layer

LLM agents handle intent ambiguity, context switching, workflow ambiguity, and optional report wording. They do not invent engineering requirements or deterministic workflow prompt text.

Detail: [`6. ai_agent_design.md`](6.%20ai_agent_design.md).

# Report vs live UI

| Pipeline | Components |
| --- | --- |

| **Live workflow UI** | Messaging + Flow Guidance (`GuidanceResolver`, `ResponseComposer`) |

Flow Guidance is **not** part of report generation. See [`10. report_generation_design.md`](10.%20report_generation_design.md) and [`docs/rules.md`](../rules.md) §21.

# Final principle

Graph defines what must happen on the active path. Validation defines what is allowed. Planner orders what to ask next. Agents navigate; calculations produce truth. Reports explain what happened.
