# Task outputs authority registry

> **EXPLANATORY_DOCUMENTATION** — Runtime authority registry. Index: [`docs/core/README.md`](README.md).

Lightweight reference for `Task.outputs` and related runtime fields. Prevents cache, projection, and debug telemetry from being treated as navigation authority.

**Navigation authority (Phase 1):** `task.outputs["engineering_plan"]` via Planner. Facts live in `execution_context.fact_store`.

## Authoritative

| Key / store | Owner | Role |
|-------------|-------|------|
| `engineering_plan` | Planner | Sole navigation authority: `next_fields`, `submittable_fields`, requirement statuses |
| `execution_context.fact_store` | Engine | User-confirmed parameter values |
| `_execution_trace` | Executor | Persisted execution history and resolved graph nodes |
| `execution_context.validation` | Validation engine | Compliance state |
| `task.status` | Execution lifecycle | Task terminal state (`completed`, `awaiting_input`, …) |
| Graph metadata written by planning refresh | Graph engine | `selected_nodes`, `path_decision`, `phase_allowed_fields`, `planning_structure_signature` |

## Projection (derived; not navigation authority)

| Key / surface | Derived from | Role |
|---------------|--------------|------|
| `goal_store` | `engineering_plan` | Compatibility projection; synced via `sync_plan_projections` |
| `graph_navigation` | `engineering_plan` | Phase buckets and active field snapshot |
| Timeline rows / step status | Plan + facts | API presentation in `api/workflow_timeline.py`, `api/serializers.py` |
| `workflow_state` (API) | Task + trace + plan projections | Desktop workflow visualization |
| `planning_projection()` dict | Plan (+ goal_store fallback for unsupported workflows) | API read model |

## Debug / telemetry only

| Key / surface | Owner | Must not affect |
|---------------|-------|-----------------|
| `TaskStateManager._step_progress` | Executor (in-memory) | `current_ask`, submittable, visited nodes, completion, readiness |
| `debug.step_progress` (API full mode) | Serializer | Interactive navigation |
| `planner_inspector_summary` | Planner inspector | Navigation gates |
| `engineering_plan_view` | Planner inspector | Navigation gates |

## Cache (recomputable; persistence cleanup deferred)

| Key | Rebuild from |
|-----|--------------|
| `engineering_plan_view` | `engineering_plan` |
| `graph_navigation` | `engineering_plan` |
| `planner_inspector_summary` | `engineering_plan` |

## Rules for new code

1. Do not use `task.outputs` calculation keys (`allowable_stress`, `required_thickness`, `mawp`, …) for timeline input **done** or navigation gates.
2. Do not read `goal_store` for `current_ask` or `submittable_fields` when `engineering_plan` exists.
3. Do not use `step_progress` for `visited_nodes` or workflow readiness.
4. Do not use `graph_navigation` snapshot as execution readiness authority — derive from `engineering_plan` at read time.
5. Do not use `phase_missing` or `revealed_input_ids` phase-bucket logic as gatherability authority when `engineering_plan` exists.
