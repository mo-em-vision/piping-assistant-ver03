# Graph Engine Design

## Purpose

The Graph Engine answers: **what must happen on the active path?** It owns expansion, reachability, branch activation, dependencies, and execution ordering. It does not validate compliance, execute formulas, or perform lookups.

Compliance: `ValidationEngine`. Calculations: `Executor`.

---

# Owns / must not own

| Owns | Must not |
| --- | --- |
| `expand_workflow`, active subgraph | Formula evaluation, lookup resolution |
| Edge `when`, applicability, expansion gates (`engine/graph/conditions.py`) | Compliance gate (`ValidationEngine`) |
| `required_user_inputs()` on active path | User-facing narration (`GuidanceResolver`) |
| `ExecutionPlan` structure (nodes, order) | Persisted task state (`State Manager`) |
| Graph version / pack compile consumption | Prompt copy |

---

# Incremental expansion lifecycle

The Graph Engine does **not** build the complete execution graph before any input is collected.

1. Select workflow.
2. `expand_workflow` with current Facts.
3. Expose unresolved expansion gates or branch decisions only.
4. Collect permitted input; re-expand.
5. Repeat until active subgraph and executable work are resolved.
6. Produce / refresh `ExecutionPlan` for ready nodes.

A partially expanded graph is not the same as a resolved executable plan.

# Condition ownership

| Concern | Owner |
| --- | --- |
| Edge `when`, applicability, expansion gates during expansion | `engine/graph/conditions.py` (`when_clause_matches`) |
| Narrow runtime expression conditions during node execution | `RuleEngine` (delegated from runners) |
| Compliance gate | `ValidationEngine` |

Components do not independently re-evaluate the same edge condition for different purposes.
